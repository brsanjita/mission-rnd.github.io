1)LeaderBoard Generator .

 

Input :

You will be writing a node script which will generate a leaderboard file from the data files . All the input and output files are csv files . Lines are seperated by \n .

 

There are 3 types of data files :
Users.csv file which contains all users data
lesson1.csv file which contains scores students got in lesson1
lesson2.csv file which contains scores students got in lesson 2
There will be only 2 lessons.

 

Users.csv file line format :

id,name,githubId,college,gender

 

Sample users.csv file line :

1,abhi,abhijith,stmartins,male

Lesson Scores line format
githubId,score.

Ex :

If lesson1.csv has
abhijith,99

It means that a student with githubID as abhijith has got 99 in lesson1 .

Output :

You will passed a path to the folder where all input files will be .In the same folder you need to generate output files too .

Output File 1 name : leaderboardsimple.csv

For the above file you will be clubbing data from users file and scores from only lesson1 file .Total marks will be same as lesson1 marks.

Small leaderboard line format :

name,college,gender,lesson1marks,totalmarks

First line in the small leaderboard should be (header line)

name,college,gender,lesson1,total

Following lines should be the students data

Output File 2 name : leaderboardlarge.csv

For the above file you will be clubbing data from users file and scores from both  lesson1 and lesson2 file .Total marks will be sum of lesson1 score and lesson2 score.

Large leaderboard line format:

Name,college,gender,test1marks,test2marks,totalmarks

First line in the large leaderboard should be  (headerline)

name,college,gender,lesson1,lesson2,total

Following lines should be the students data

Note : Look into the testcase1 ,testcase 2 folders for sample input files /test files and ExpectedLeaderboardSmall and ExpectedLeaderBoardLarge files and UNDERSTAND HOW YOU NEED TO FORM AND WRITE THE DATA .

Methods :

 

To the 2 functions you will be passed a path to a folder which will have the 3 files ,  users,lesson1 and lesson2 .You need to generate the two leaderboard files ,leaderboardsmall.csv and leaderboardlarge.csv in the same directory

 

generateLeaderBoardSmall(folderPath,callback) :  Generates small leaderboard file and calls callback

generateLeaderBoardLarge(folderPath,callback) :  Generates large ledaerboard file and calls callback

 

Sample Input Files

users.csv

1,abhijith,github100,stmartins,male

2,nikhil,github101,cvr,male

 

lesson1.csv

github100,90


lesson2.csv

github100,1
github101,99

 

Sample Output Files


leaderboardsmall.csv

name,college,gender,lesson1,total

abhijith,stmartins,male,90,90

nikhil,cvr,male,0,0

 

leaderboardlarge.csv

 

name,college,gender,lesson1,lesson2,total

abhijith,stmartins,male,90,1,91

nikhil,cvr,male,0,99,99

2)RestaurantFinder

Problem :
You will be writing a Express Server which will help in finding,adding,deleting restaurants etc.
 

Restaurant Model : (id,name,phone,budget,cuisine,colony,city) .

Constraints :


Name should be a string less than 30 chars , which consists of only letters,digits and spaces.
Cities can be hyderabad,vizag or bhimavaram only or any of its capital letter forms. [HyDeRaBAD is also valid ].
Colonies can be anything , but less than 30 characters .
Phone number must be of 10 characters ,which consists of only numbers

Budget must be a positive integer greater than 50 .
Cuisine can be anything , but less than 30 characters .(Should contain only letters ,No spaces or digits)

If the request doesn't match any of the above constraints , return 400 as statusCode.
In the whole project ,every value (Except Name) can be of either lowercase or upper case and both are same.like northindian is same as NorthIndian. so you can store everything in lowercase in DB .
For names , Chutneys is different than chutNeys.
